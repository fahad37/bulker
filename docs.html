Documentation

This guide covers setup, usage, deployment, and troubleshooting.

CSV Format

  • Headers: email,name
  • Saved as UTF-8 CSV
  • Example rows: alice@example.com,Alice

Personalization

  • Use placeholders like {{name}} in subject and message
  • Missing fields render as empty strings

Sending via EmailJS

  1. Create an EmailJS account and add an Email Service
  2. Create an email Template with variables: to_email, to_name, subject, message, from_name
  3. Copy your Public Key, Service ID, and Template ID
  4. In the app, select EmailJS and paste these values
  5. Use Send Test before bulk sending

Direct SMTP

  • Enter SMTP host, username (your email), and an app password
  • Set port 465 or 587. Some providers block direct SMTP from browsers
  • If sending fails, prefer EmailJS or a server-side relay

GitHub Pages Deployment

  1. Create a new repository on GitHub
  2. Add index.html, styles.css, app.js, and this docs.html
  3. Commit and push
  4. Open Settings → Pages → Build and deployment → Source: Deploy from a branch; Branch: main / root
  5. Wait for Pages to publish and open the provided URL

Live Server

  • Upload the folder to any static host (Netlify, Vercel, shared hosting)
  • Ensure all files are in the same directory

Troubleshooting

  • CSV fails to parse: ensure headers are exactly email,name; save as CSV
  • EmailJS fails: verify public key, service id, template id; check allowed domains
  • SMTP fails: confirm app password is active; some providers disallow browser SMTP
  • Rate limits: increase Send Delay or send in smaller batches

Security Notes

  • Credentials are used in-memory and not stored by the app
  • Use per-app passwords and avoid shared devices
